# 日语教学平台一期最终功能边界清单

## 1. 文档目的

本清单用于明确一期 MVP 的最终功能边界，作为以下工作的直接输入：

- 原型设计
- 开发排期
- 报价评估
- 测试范围
- 上线验收

本清单以“能形成业务闭环、能落地运营、能控制复杂度”为原则。

## 2. 边界分层

### 2.1 P0 必须做

一期必须上线，否则核心业务闭环不成立。

### 2.2 P1 建议做

建议一期实现，能明显提升运营和产品完整度，但如果时间不足可后移。

### 2.3 P2 明确不在一期做

一期不做，避免范围膨胀。

## 13. 一期验收标准

一期至少应能完整跑通以下业务链路：

1. 用户通过公众号了解项目并提交咨询
2. 用户在小程序购买会员或课程
3. 用户进入课表并参与课程学习
4. 教师/运营在后台完成课程、班级、课次、作业、活动管理
5. 学员完成每日任务和作业提交
6. 教师完成反馈，系统形成学习档案
7. 系统生成提醒与跟进任务
8. 后台可查看订单、退款、核心报表
