# 日语教学平台页面结构与信息架构清单

## 1. 文档目的

本清单用于明确一期 MVP 的页面结构、页面职责和信息架构边界，作为以下工作的输入：

- 原型设计
- 页面优先级排序
- 前端开发拆分
- 接口设计对齐

## 2. 端划分

一期页面分为三端：

- 公众号承接端
- 小程序学员端
- Web 后台端

## 3. 公众号承接端

### 3.1 页面结构

#### 首页

作用：

- 展示品牌定位
- 传达平台核心价值
- 引导用户咨询或进入小程序

核心信息：

- 平台介绍
- 日语教学服务特色
- 目标人群
- 课程入口
- 咨询入口
- 小程序跳转入口

#### 课程介绍页

作用：

- 展示基础课程与精品课程
- 帮助用户理解课程区别

核心信息：

- 课程分类
- 适合对象
- 学习周期
- 上课形式
- 服务内容
- 购买或咨询引导

#### 教师介绍页

作用：

- 展示教师背景与教学风格
- 增强用户信任

核心信息：

- 教师头像
- 教师简介
- 擅长方向
- 授课特色

#### 活动宣传页

作用：

- 展示公开活动和会员活动
- 承接品牌获客

核心信息：

- 活动名称
- 活动时间地点
- 活动亮点
- 报名引导

#### 咨询表单页

作用：

- 收集用户线索
- 进入后台咨询线索池

核心字段建议：

- 姓名
- 联系方式
- 当前水平
- 学习目的
- 意向课程
- 备注

## 4. 小程序学员端

### 4.1 顶层信息架构

建议一级结构：

- 首页
- 课程
- 任务
- 活动
- 我的

### 4.2 页面清单

#### 首页

作用：

- 展示当前用户最重要的信息
- 承接到课表、任务、作业、活动

核心模块：

- 当前会员状态
- 当前课程/班级
- 最近课表
- 今日任务入口
- 待完成作业
- 活动推荐
- 消息提醒摘要

#### 课程列表页

作用：

- 展示可了解或已购买的课程

核心模块：

- 基础课程区域
- 精品课程区域
- 课程卡片
- 购买状态
- 课程详情入口

#### 课程详情页

作用：

- 展示课程具体内容和购买信息

核心模块：

- 课程简介
- 适合对象
- 周期与形式
- 教学服务内容
- 是否包含会员权益
- 购买按钮

#### 支付页

作用：

- 完成会员、课程、活动购买

核心模块：

- 商品信息
- 订单金额
- 支付方式
- 支付结果反馈

#### 我的课表页

作用：

- 查看已报名班级的课次安排

核心模块：

- 班级信息
- 课次时间
- 教师信息
- 请假入口
- 补课/调课状态提示

#### 作业页

作用：

- 查看待完成与已完成作业

核心模块：

- 待提交作业
- 已提交作业
- 作业详情
- 提交入口
- 教师反馈入口

#### 作业提交页

作用：

- 完成文字、图片、音频提交

核心模块：

- 作业内容
- 截止时间
- 文本输入
- 图片上传
- 音频上传
- 提交状态

#### 每日任务页

作用：

- 承接每日打卡任务

核心模块：

- 今日任务类型
- 今日题量
- 完成进度
- 提交结果
- 连续打卡天数

#### 活动列表页

作用：

- 展示可报名活动

核心模块：

- 活动卡片
- 公开/会员标识
- 免费/收费标识
- 报名状态

#### 活动详情页

作用：

- 展示活动详情并完成报名

核心模块：

- 活动说明
- 时间地点
- 费用说明
- 报名按钮
- 取消报名入口（如已报名）

#### 学习档案页

作用：

- 展示学习过程数据与反馈

核心模块：

- 会员状态
- 所属课程/班级
- 到课记录
- 作业完成情况
- 每日任务完成情况
- 教师反馈
- 阶段性总结

#### 消息提醒页

作用：

- 集中查看提醒消息

核心模块：

- 上课提醒
- 作业提醒
- 每日任务提醒
- 会员到期提醒

#### 个人中心页

作用：

- 管理个人信息和会员信息

核心模块：

- 基础资料
- 会员状态
- 我的订单
- 我的活动
- 设置入口

## 5. Web 后台端

建议一级菜单：

- 仪表盘

- 活动
- 订单与退款
- 报表
- 系统设置

## 6. 页面优先级建议

### 第一批优先原型

- 公众号首页
- 咨询表单页
- 小程序首页
- 小程序课程详情页
- 小程序支付页
- 小程序课表页
- 小程序作业页
- 小程序每日任务页
- 小程序学习档案页
- 后台登录页
- 后台仪表盘
- 学员列表/详情页
- 课程列表/编辑页
- 班级列表/详情页
- 作业资源库页
- 作业批改页
- 订单列表页

### 第二批原型

- 活动列表/详情页
- 会员方案与权益配置页
- 退款处理页
- 报表页
- 系统设置页

## 7. 信息架构原则

- 公众号负责展示和承接，不承担学习主流程
- 小程序负责学员主流程，不承担复杂管理流程
- Web 后台负责教师、教务、运营、管理员的管理流程
- 同一类业务尽量只在一个端完成，避免重复入口
- 一期按最小闭环组织页面，不为未来复杂扩展过早设计过多层级
